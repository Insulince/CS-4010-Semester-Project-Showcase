I have two Servlets, a session servlet and a controller servlet. I mapped the Session Servlet to the blank string pattern because this is the only servlet that I want tp be publicly facing. This means you can hit "http://localhost:8080/j-cubed" (or ".../j-cubed/") directly (with not following url pattern) and you will be taken to the Session servlet. The controller servlet is mapped to /controller, however it can't be hit publicly without the "Authorized" attribute being set on the request beforehand, which can only be done by the session servlet and being validated first.
